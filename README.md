# Integrated mRNA Sequence Optimization

This repository contains the one-step full-length mRNA generation code for the method described in Gong et al., *Integrated mRNA sequence optimization using deep learning* (Briefings in Bioinformatics, 2023). Given a protein sequence, the pipeline first optimizes the coding sequence (CDS) with a BiLSTM-CRF codon-selection model, then generates matching 5' UTR and 3' UTR sequences with two fine-tuned mBART models. The final output is a complete transcript design represented as:

```text
(5' UTR, optimized CDS, 3' UTR)
```

The CDS and UTR modules are integrated in `main.py`, so users can run full-length generation from an amino-acid input file with one command.

## Repository Contents

```text
main.py                         # one-step full-length inference entry point
UTR.py                          # 5' UTR / 3' UTR generation with mBART models
demo.decode.config              # example decoding configuration
demo.train.config               # legacy CDS training configuration template
sample_data/                    # example amino-acid input and CDS output
model/, utils/                  # BiLSTM-CRF codon optimization code
Codon_Tokenizer_text_bs/        # tokenizer used by the UTR generators
mbart-iitb-cc-5u-fbs/           # 5' UTR model directory
mbart-iitb-cc-3u-fbs/           # 3' UTR model directory
lstmcrf.dset                    # CDS vocabulary/data dictionary artifact
lstmcrf.113.model               # trained CDS model artifact
```

## Requirements

The code uses PyTorch for the BiLSTM-CRF CDS optimizer and Hugging Face Transformers for the mBART UTR generators. The core dependencies, based on the `bart_old` environment used for this project, are:

```bash
python == 3.8.20
torch == 2.4.1
transformers == 4.18.0
tokenizers == 0.12.1
numpy == 1.24.4
sacremoses == 0.1.1
```

Example installation:

```bash
pip install "torch==2.4.1" "transformers==4.18.0" "tokenizers==0.12.1" "numpy==1.24.4" "sacremoses==0.1.1"
```

If you use CUDA, install the PyTorch build matching your local driver/toolkit. The tested `bart_old` environment used `torch==2.4.1+cu121`.

## Model Artifacts

Full inference requires the following pretrained artifacts to be present at the paths referenced by `demo.decode.config` and `main.py`:

```text
lstmcrf.dset
lstmcrf.113.model
Codon_Tokenizer_text_bs/
mbart-iitb-cc-5u-fbs/
mbart-iitb-cc-3u-fbs/
```

For public release, large binary files may be distributed separately, for example through Git LFS or a release download. Before running inference, make sure each model directory contains a valid `pytorch_model.bin` and that `lstmcrf.dset` / `lstmcrf.113.model` are not empty placeholder files.

## Quick Start

1. Put one or more amino-acid sequences in a plain text file, one sequence per line:

```text
MAEPRQEFEVMEDHAGTYGLGDRKDQGGYTMHQDQEGDTDAGLKESPLQTPT...
```

The example input is:

```text
sample_data/mDS-20230925.txt
```

2. Check `demo.decode.config`:

```text
status=decode
raw_dir=sample_data/mDS-20230925.txt
nbest=1
decode_dir=sample_data/mDS_test.out
dset_dir=./lstmcrf.dset
load_model_dir=./lstmcrf.113.model
```

3. Run full-length generation from the repository root:

```bash
python main.py --config demo.decode.config
```

The optimized CDS is written to `decode_dir`. The complete design is printed by `main.py` as a Python tuple:

```text
('generated_5UTR_sequence', 'optimized_CDS_sequence', 'generated_3UTR_sequence')
```

Sequences are represented with DNA bases (`A`, `T`, `C`, `G`). The CDS model appends a stop codon during decoding through the terminal amino-acid symbol used internally by the code.

## Input and Output Format

### Input

`raw_dir` should point to a text file containing amino-acid sequences:

```text
PROTEIN_SEQUENCE_1
PROTEIN_SEQUENCE_2
...
```

Do not include FASTA headers in the default inference path. If your data are in FASTA format, convert them to one raw amino-acid sequence per line before running `main.py`.

### Intermediate CDS Output

`decode_dir` stores the optimized coding sequence generated by the BiLSTM-CRF module:

```text
ATGGCAGAGCCCAGG...
```

### Full-Length Output

The UTR module consumes the optimized CDS and returns:

```text
5' UTR + optimized CDS + 3' UTR
```

In the current `main.py`, this is printed as a tuple rather than written to a separate file. For batch UTR generation from a CDS file, see `get_full_sequence()` in `UTR.py`.

## Method Summary

The pipeline follows the integrated design idea of Gong et al.:

- CDS optimization is formulated as sequence labeling. Each amino acid is assigned a codon-box label, and `utils/codonBox.py` converts amino-acid/label pairs back to codons.
- The CDS sequence is passed to separate encoder-decoder models for 5' UTR and 3' UTR generation.
- `Codon_Tokenizer_text_bs` tokenizes codon-box representations for the mBART-based UTR models.

## Training

`demo.train.config` is retained as a legacy CDS training template for the BiLSTM-CRF component. The one-step full-length pipeline in `main.py` is primarily intended for decoding with pretrained CDS and UTR models. If retraining is needed, prepare CDS training/dev/test files in the NCRF++-style amino-acid plus codon-box label format and update:

```text
train_dir=
dev_dir=
test_dir=
model_dir=
word_emb_dir=
```

The UTR models are not trained through `main.py`; they are loaded as Hugging Face model directories by `UTR.py`.
